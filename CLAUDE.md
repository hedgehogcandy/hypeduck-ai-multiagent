# 🦆 HypeDuck AI Multi-Agent System

> 인스타그램 계정 육성 및 자동화를 위한 AI 멀티에이전트 시스템

---

## 🎯 시스템 목표

1. **계정 육성**: 알고리즘에 최적화된 계정 성장
2. **트렌드 소싱**: 잘 터진 영상/콘텐츠 자동 발굴
3. **경쟁사 트래킹**: 레퍼런스 채널 실시간 모니터링
4. **자동화 파이프라인**: 소싱 → 스크립트 → 영상 제작 자동화

## 📋 워크플로우

### Phase 1: 레퍼런스 채널 수집

```
1. 회원님을 위한 추천 → 카테고리별 확인
2. 알고리즘 피드 추천 분석
3. 경쟁 채널 식별 및 저장
```

### Phase 2: 잘 터진 영상 추리기

```
기준: 조회수 대비 댓글 비율
  - 10:1 → BEST 🏆
  - 20:1 → GOOD 👍
  - 30:1 → OK 👌

확인 방법:
  - 고정 댓글 확인
  - 플러그인으로 댓글 분석
  - engagement rate 계산
```

### Phase 3: 스크립트 추출 및 분석

```
1. https://www.dictationer.com/ko 로 자막 추출
2. 대본 구조 파악
   - 10:1 비율 영상들의 구조 템플릿화
   - 제품별 맞는 글 구조 매칭
3. 지식 창고에 저장
```

### Phase 4: 차별화 포인트 분석

```
분석 요소:
  - 폰트, 썸네일 스타일
  - 효과음 패턴
  - 후킹 방식
  - TTS 특징
  - 편집 스타일
```

### Phase 5: 트래킹 시스템

```
저장 대상:
  - 레퍼런스 계정 목록
  - 잘 터진 영상 아카이브
  - 스크립트 템플릿
  - 내 계정 ↔ 레퍼런스 연결 관계
```

## 🤖 에이전트 역할

### 1. Researcher (리서처)
- 트렌드 분석
- 레퍼런스 채널 발굴
- 잘 터진 영상 식별

### 2. Analyst (분석가)
- 영상 구조 분석
- 댓글 비율 계산
- 차별화 포인트 추출

### 3. Writer (작가)
- 스크립트 템플릿 적용
- 대본 재작성
- 후킹 문구 생성

### 4. Sourcer (소서)
- 영상 소스 찾기
- 이미지 검색
- 상품 매칭

### 5. Tracker (트래커)
- 계정 모니터링
- 변화 감지
- 알림 발송

## ⚠️ 주의사항

1. **계정 안전 최우선**
   - Rate Limiting 필수
   - 사람처럼 행동
   - 의심스러운 패턴 금지

2. **데이터 처리 분리**
   - Chrome MCP: 조작만
   - Backend: 파싱/분석

3. **페르소나 일관성**
   - 각 계정별 캐릭터 유지
   - 알고리즘 학습 데이터 축적
